# DeskBot-Discover

**您的AI驱动音频智能伴侣，助您享受档案级音乐发现**

DeskBot-Discover是一款模块化、无服务器的配套工具，旨在跨多个流媒体目录交叉识别高分辨率音频轨道——无需身份验证或订阅密钥。它在后台静默运行，匹配元数据、指纹和发行标识符，以显示任何曲目、专辑或播放列表的最高码率版本。

与传统搜索工具不同，DeskBot-Discover不存储、缓存或重新分发音频文件。相反，它作为公共目录索引之间的知识桥梁，使用户能够明智地决定从哪里获取录音室级母带。该系统专为档案管理员、广播节目制作人、发烧友研究员和音乐图书馆员设计，他们需要关于音频保真度的确定性答案——而非流媒体便利性。

DeskBot-Discover纯粹是一个研究和识别工具。它不会绕过地区限制，不会违反服务条款，也不会允许未经授权的下载。所有操作均为只读、基于查询的，并设计以符合适用版权框架下的合理使用和数据挖掘豁免。

## 概述

现代音乐生态系统已经碎片化。同一首曲目可能在一个平台上以有损Ogg Vorbis格式提供，在另一个平台上以AAC 256kbps提供，而在精品高分辨率音乐商店中则以真正的24-bit/96kHz FLAC格式提供。识别哪个版本在哪个平台可用——而无需手动创建账户、安装专有应用程序或泄露个人数据——这一挑战历来需要深厚的目录知识或昂贵的聚合服务。

DeskBot-Discover通过提供一个轻量级、可查询的界面解决了这个问题，该界面能够查询Tidal、Qobuz、Amazon Music、Deezer等平台的公开目录端点。它通过交叉引用ISRC代码、UPC条形码、专辑发行ID和嵌入式声学指纹，返回一个按位深度、采样率和编解码器完整性排序的可用源格式列表。

无需创建账户。没有用户追踪。没有任何音频数据被写入磁盘。DeskBot-Discover是一个纯粹的元数据智能层——透视流媒体目录底层基础设施的透镜。

## 核心功能

DeskBot-Discover有三种运行模式：

### 1. 🎯 **单曲解析**
提供Spotify曲目URL、ISRC代码或 `艺人 + 曲目标题` 字符串。DeskBot-Discover返回一个排名矩阵，显示哪些流媒体平台提供该曲目、使用什么码率/编解码器，以及是否存在高分辨率（≥24-bit/96kHz）母带。

### 2. 📋 **播放列表与专辑发现**
提供公开播放列表链接或专辑UPC。DeskBot-Discover遍历每一首曲目，执行单独解析，并以JSON、CSV或人类可读表格格式编译保真度报告。

### 3. 🔍 **目录审核**
对于本地音乐库中已知缺失或质量退化的曲目，DeskBot-Discover可以扫描目录，识别在另一个发行ID或厂牌下是否存在更高质量的版本。适用于音乐库策展和用录音室级原版替换有损副本。

## 架构理念

DeskBot-Discover遵循 **[R2D2 框架](https://en.wikipedia.org/wiki/Read-only_memory)** — **只读（Read-only）**, **弹性（Resilient）**, **确定性（Deterministic）**, **可发现（Discoverable）**, **独立（Independent）**.

1. **只读** – 不写入任何外部服务，磁盘上不持久化任何数据
2. **弹性** – 如果某个源无法访问，优雅地失败并使用回退启发式方法
3. **确定性** – 给定相同输入，保证相同输出
4. **可发现** – 所有输出包含来源标签，显示每个元数据点的具体来源
5. **独立** – 无单点故障；每个查询源都是解耦的

这意味着DeskBot-Discover可以安全地部署在需要审计网络活动的环境中，或在气隙系统上通过手动数据注入运行。

DeskBot-Discover以单个Python模块形式分发，除标准库外没有外部运行时依赖。

1. 将 `deskbot_discover.py` 模块放在你的工作目录或Python路径中
2. 通过测试查询验证激活： 
   `python -c "from deskbot_discover import resolve; print(resolve('track', '4cOdK2wGLETKBW3PvgPWqT'))"`
3. 预期输出: 包含匹配来源及格式详细信息的JSON数据

### 快速测试

```python
from deskbot_discover import resolve

result = resolve(source='spotify', identifier='4cOdK2wGLETKBW3PvgPWqT')  
print(result['formats']['highest_res']['provider']) # 例如：'Qobuz'
```

没有配置文件。没有环境变量。没有密钥。

## 使用场景

### 🏛️ 音乐库策展

你有一个3万首曲目的FLAC库。你怀疑其中一些文件是从MP3源升频而来。对库进行批量审计，找出哪些曲目存在来自验证来源的真正高分辨率版本。

### 🎧 电台质量保证

社区电台经常从有损流中获取音频。使用DeskBot-Discover生成报告，显示播放列表中哪些曲目有已知的高分辨率母带，然后用适当的源素材替换电台副本。

### 📀 黑胶翻录验证

你数字化了一个黑胶收藏。DeskBot-Discover可以将翻录的频谱指纹与已知数字母带进行比对，确认黑胶源是来自高分辨率母带压制，还是CD时代的转录。

### 🔬 学术音乐学研究

研究响度战争、动态范围压缩或重制版趋势的研究人员，可以利用DeskBot-Discover系统性地整理同一张专辑数十年来不同版本的母带信息。

## 24/7 后台运行

DeskBot-Discover设计用于无头环境中的无人值守运行：

- **自动限速** – 以指数回退方式尊重源速率限制
- **恢复逻辑** – 如果播放列表扫描中途中断，保存进度并从最后一次成功查询处恢复
- **日志记录** – 结构化JSON日志输出到标准输出或文件，日志级别可配置
- **健康检查** – 定期报告各源可用状态

适合cron定时任务的夜间目录审计示例：

```python
from deskbot_discover import batch_resolve
playlist_id = '37i9dQZF1DXcBWIGoYBM5M'
report = batch_resolve(source='spotify_playlist', identifier=playlist_id)
with open('audit_2026.json', 'w') as f:
    f.write(report.to_json())
```

## 用例：档案与研究

对于数字保存工作者来说，知道曲目存在高分辨率版本的*哪个版本*可以判断该源是否适合长期存档。DeskBot-Discover为每个匹配来源提供**保真度评分** (0.0 – 1.0)，计算公式如下：

- 位深度 （权重最高：24-bit > 16-bit > 8-bit）
- 采样率 （权重最高：192kHz > 96kHz > 44.1kHz > 更低）
- 容器完整性 （FLAC/ALAC > WAV/AIFF > 有损格式）
- 来源透明度 （源是否提供母带元数据而非编码猜测）

该评分帮助档案管理员快速筛选哪些来源需要进一步人工检查。

## 限制与免责声明

**重要：DeskBot-Discover是一款研究级元数据工具。它不用于规避平台条款、未经授权复制或重新分发受版权保护的材料。**

1. **无音频提取** – DeskBot-Discover从不下载、流式传输或捕获音频数据。它只读取来自公开API的文本元数据。
2. **目录不一致** – 平台会定期删除、重新添加或重新编码曲目。源上的可用性不保证始终有效。
3. **地区差异** – 某些高分辨率版本在元数据中可能存在区域锁定。DeskBot-Discover报告的是API对源的公开端点返回的内容；这可能与实际的地理位置可用性不同。
4. **指纹准确度** – 对于非常短的曲目（<30秒）或静音较多的录音，声学指纹匹配的误报率约为1%。
5. **速率限制** – 激进的播放列表扫描（>500 首曲目）可能会在某些源触发临时IP封锁。DeskBot-Discover内置了延迟机制，但用户需负责尊重合理使用。
